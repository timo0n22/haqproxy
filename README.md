# hackproxy — свой Burp/Caido без лимитов бесплатного тарифа

Личный intercepting-proxy: TLS-перехват на базе mitmproxy (не переизобретаем этот
кусок), а History + Repeater — свои, без лимитов на вкладки/потоки, как в
бесплатных тарифах Burp/Caido.

Работает на MacBook (роль "атакующей машины" в вашей схеме) — годится и против
домашней лабы на Pi, и против реальных bug bounty целей.

## Состав v1 (MVP)

- **Proxy + History** — весь HTTP(S)-трафик браузера пишется в SQLite сырыми
  байтами (заголовки — с сохранением порядка и дублей).
- **Repeater** — сколько угодно вкладок, ручное редактирование сырого запроса
  байт-в-байт (не через `requests`/`curl`, которые нормализуют заголовки —
  здесь свой сокет-клиент, важно для request smuggling / WAF-байпасов).
- **Scope** — список хостов для фильтрации истории.

Пока нет (следующие итерации): Intruder-аналог, свой Collaborator на VPS,
Match & Replace, WebSocket-перехват, Decoder.

## Установка

```bash
cd hackproxy
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

## Первый запуск

**1. Поднять MITM-прокси** (в одном терминале):

```bash
mitmdump -s recorder.py -p 8080
```

При первом запуске mitmproxy сам создаст CA-сертификат в `~/.mitmproxy/`.

**2. Установить сертификат в доверенные** — один раз:
   - macOS: открыть `~/.mitmproxy/mitmproxy-ca-cert.pem` в Keychain Access,
     найти сертификат "mitmproxy", включить доверие "Always Trust" для SSL.
   - Либо доверять точечно только в тестовом браузерном профиле
     (Firefox, например, использует свой собственный набор доверенных
     сертификатов, не системный — это удобно, чтобы не трогать доверие
     во всей системе, только в браузере для тестов).

**3. Настроить браузер на использование прокси** `127.0.0.1:8080` (HTTP и HTTPS).
   Проще всего — через отдельный профиль браузера, чтобы не проксировать
   вообще весь свой обычный трафик.

**4. Поднять веб-интерфейс** (в другом терминале):

```bash
python3 app.py
```

Открыть `http://127.0.0.1:5050` в браузере (в обычном профиле, НЕ через
сам прокси — иначе будете проксировать самого себя).

## Как пользоваться

- Ходите браузером (через профиль с настроенным прокси) по целям —
  всё появляется в History слева, обновляется каждые 4 секунды.
- Клик по строке в History → открывается новая вкладка Repeater с этим
  запросом, можно редактировать сырой текст и пересылать сколько угодно раз.
- "+" на панели вкладок — создать пустую вкладку Repeater с нуля.
- Scope (шапка) — добавляете паттерн вроде `*.example.com`, фильтр
  "только scope" в History показывает только то, что попадает под паттерн.

## Известные ограничения v1

- Chunked-ответы детектируются эвристикой по завершающему `0\r\n\r\n` —
  не полный RFC-разбор trailers/расширений чанков (в большинстве случаев
  не критично).
- Тела запросов/ответов хранятся как текст через latin-1 (lossless для
  байт), но большие бинарные тела (файлы, картинки) будут раздувать
  размер БД — если понадобится, добавим отдельное хранение по хэшу.
- TLS-проверка сертификата цели в Repeater по умолчанию выключена
  (`verify_tls: false`) — это осознанный выбор для тестирования
  самоподписанных/staging-целей; включайте чекбокс "проверять сертификат"
  для боевых целей, если это важно для конкретного теста.
